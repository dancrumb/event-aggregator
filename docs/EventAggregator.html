EventAggregator

onAll(eventName, listener)

Source:
  • index.js, line 223

Once all of the emitters in this aggregator have emitted a eventName event, the associated listener is triggered.

Like EventEmitter#on, this will continue to fire until it is explicitly removed.

The listener will receive an array of the arguments from each of the events that were emitted from the aggregated emitters. The array is in the order in which the emitters were added to this aggregator.

If an emitter has fired multiple times, the listener will get the arguments from the first event.

Once the listener has been triggered, this aggregator is reset for this event.

removeEmitter(emitter)

Source:
  • index.js, line 159

Removers an emitter from the group.

Any listeners attached to this emitter are also removed.

If any listeners are attached via onAll or onceAll, then removing this emitter may result in those listeners being fired. This will happen if the emitter that is being removed is the only one in the group that hadn't fired.

Put another way, if all of the emitters except this one had fired, then removing this one fires the onAll and onceAll listeners.

If there are listeners that are waiting for all of the emitters in this aggregation to emit, then calling this method will reset the aggregator so that it's as if none of them have emitted.

If an eventName is provided, this is limited to listeners for that event. Otherwise, calling this resets all of the listeners.

setMaxListeners(max) → {EventAggregator}

Source:
  • index.js, line 302

By default EventEmitters will print a warning if more than 10 listeners are added for a particular event. This is a useful default that helps finding memory leaks. Obviously, not all events should be limited to just 10 listeners. The emitter.setMaxListeners() method allows the limit to be modified for this specific EventEmitter instance.

The value can be set to Infinity (or 0) to indicate an unlimited number of listeners.
